# Vampire Crawlers Mod 开发记录

## 项目环境

- 游戏目录通过仓库根目录的 `game/` 指向 `C:\Programs\Steam\steamapps\common\Vampire Crawlers`。
- BepInEx 6 IL2CPP 已放入游戏目录并运行过游戏，interop 程序集位于 `game/BepInEx/interop/`。
- 游戏主逻辑程序集是 `Pancake.dll`，当前项目通过 `game/BepInEx/interop/Pancake.dll` 引用游戏类型。
- `DummyDll/` 是 Il2CppDumper 生成的引用程序集，`GameSourceCode/` 是反编译代码，用于查证游戏逻辑。

## 构建与部署

- 构建命令：

```powershell
dotnet build .\src\VampireCrawlersMod\VampireCrawlersMod.csproj -c Debug --no-restore
```

- 项目构建后会把 `VampireCrawlersMod.dll` 自动复制到 `game/BepInEx/plugins/`。
- 如果复制 DLL 失败，优先确认游戏是否仍在运行；游戏运行时可能锁定插件 DLL。
- 如果 dotnet 首次运行尝试写用户目录导致权限问题，可以临时设置 `DOTNET_CLI_HOME` 到项目内目录，构建后删除该临时目录。

## BepInEx 日志

- 插件日志建议只输出 ASCII 或英文，避免控制台编码不一致导致中文显示乱码。
- 当前插件启动日志使用英文：`Vampire Crawlers Mod 0.1.0 loaded`。

## IL2CPP 注意事项

- 托管类型判断不能完全依赖 C# 的 `is`。例如 `CardCostType` 的运行时对象可能被包装成基类代理，`costType is WildCostType` 可能不命中。
- 判断 IL2CPP 实际子类型时优先使用 `TryCast<T>()`，例如万能牌费用类型：

```csharp
costType is WildCostType || costType.TryCast<WildCostType>() != null
```

- 新增需要挂到 Unity 对象上的 `MonoBehaviour` 类型要先通过 `ClassInjector.RegisterTypeInIl2Cpp<T>()` 注册，再添加组件；纯静态辅助类不需要注册。
- 暴露给 Unity/IL2CPP 调用的组件构造函数需要保留 `IntPtr ptr` 构造函数。

## 输入与 UI

- 游戏启用了新版 Input System，不要在 Mod 中调用 `UnityEngine.Input`，否则会报：

```text
You are trying to read Input using the UnityEngine.Input class, but you have switched active Input handling to Input System package in Player Settings.
```

- 当前整理按钮的点击和拖动通过 `OnGUI()` 中的 `Event.current` 处理，避免直接读 `UnityEngine.Input`。
- 不要在 `Update()` 中调用 `GUI.skin` 或其他 IMGUI 绘制函数，否则会报：

```text
You can only call GUI functions from inside OnGUI.
```

- 不要使用 `GUI.DrawTexture(Texture2D.whiteTexture)` 绘制按钮，IL2CPP 下曾触发 method unstripping 相关异常。
- 暂停菜单打开时，Screen Space Overlay 按钮可能盖在菜单上；当前通过 `ModalManager.Instance.HasModalActive` 检测到 Modal 后隐藏按钮。

## 手牌整理

- 手牌整理同时处理数据顺序和显示层级：
  - `CardPileModel.TrySwapCards(...)` 调整手牌数据顺序。
  - `CardSlot.transform.SetSiblingIndex(...)` 调整 UI 槽位顺序。
  - `SortingGroup.sortingOrder` 和 `CardView/TweenContainer.SetSiblingIndex(...)` 调整遮挡层级。
  - `CardLayoutGroup.ForceLayoutRefresh()` 配合 `LayoutRebuilder.ForceRebuildLayoutImmediate(...)` 触发布局刷新。
- 只调整 siblingIndex 会导致手牌位置不规整，必须调用游戏已有的 `CardLayoutGroup` 刷新逻辑。
- 排序规则：
  - 普通牌按费用从小到大。
  - 万能费用类型牌排在最右边。
  - 同组同费用保持整理前的相对顺序。
- “万能牌”不是牌名，而是费用类型 `WildCostType`；宝石把普通牌改成万能牌时，会通过 `OverrideCardCostTypeGemEffect` 修改运行时 `CardCostType`。

## 卡牌碎裂提示

- 碎裂剩余次数显示通过 patch `CardView.GetCardDescription(...)` 追加说明文字。
- 早期尝试在卡面上额外挂 UI 文本时，选中/放大/旋转状态下容易和卡牌本体不同步；当前方案复用卡牌说明区域，更稳定。
- 只在 `BreakableCard.CrackState == CardCrackState.Cracked` 且卡牌未 `IsBroken` 时显示剩余次数。
